# 组长复现指南（必读）

> **适用对象**：组长 / 验收老师  
> **目标**：下载项目后双击启动，界面与数据库与提交者一致。  
> **不需要**：Git、PowerShell、切换分支。

---

## 推荐方式：GitHub 下载 ZIP → 双击启动

### 第 1 步：下载

1. 打开：https://github.com/Harrydai4/CampusStudyRoomReservationManagementSystem  
2. 点击绿色 **Code** → **Download ZIP**  
3. 解压到任意目录（例如 `D:\CSRRM`）

> **不必** `git clone`，**不必** `git checkout` 任何分支。  
> 默认 **master** 分支的 ZIP 已包含完整代码、数据库 SQL、图片与启动器。

### 第 2 步：进入文件夹

打开解压后的目录，确认能看到：

- `pom.xml`
- `组长一键启动.vbs`
- `docs\06-部署配置\database-full.sql`

### 第 3 步：双击启动

| 文件 | 说明 |
|------|------|
| **`组长一键启动.vbs`**（推荐） | 弹窗输入 MySQL 密码；选是否导入数据库 |
| `组长一键启动.bat` | 同上，在命令行窗口操作 |

脚本自动完成：检查环境 → 导入数据库 → 启动后端 → 打开浏览器 **http://localhost:8080**

**停止系统**：关闭标题为「CSRRM-后端服务」的窗口。

## 事先需要安装（无法通过双击自动安装）

| 项目 | 要求 |
|------|------|
| JDK | **21+**（`java -version` 可检查） |
| MySQL | **8.x**，服务已启动（一般为 MySQL80） |
| PATH | 能运行 `mysql` 命令（MySQL 的 bin 目录） |

**仅需问答输入**：本机 MySQL 的 **root 密码**（与提交者可以不同）。

## 常见问题

| 现象 | 处理 |
|------|------|
| 提示缺少 `database-full.sql` | 重新从 GitHub **Download ZIP**（不要只复制部分文件夹） |
| `Access denied for user 'root'` | 密码填**组长自己**的 MySQL 密码 |
| `mysql` 不是内部命令 | 将 MySQL `bin` 加入系统 PATH |
| 页面空、无数据 | 启动时选「是」导入数据库；或重新双击 vbs 并选导入 |
| 8080 被占用 | 见 `组长一键启动.bat` 说明或联系提交者 |
