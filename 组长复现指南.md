# 组长复现指南（必读）

> **适用对象**：组长 / 验收老师  
> **目标**：下载项目后双击启动，界面与数据库与提交者一致。  
> **不需要**：Git、PowerShell、切换分支。

---

## 推荐方式：GitHub 下载 ZIP → 双击启动

### 第 1 步：下载

**方式 A（推荐，一条链接下载完整包）**  

直接下载：  
[https://github.com/Harrydai4/CampusStudyRoomReservationManagementSystem/archive/refs/heads/v1.3-harrydai4.zip](https://github.com/Harrydai4/CampusStudyRoomReservationManagementSystem/archive/refs/heads/v1.3-harrydai4.zip)  

解压到任意目录（例如 `D:\CSRRM`）。

**方式 B**  

打开仓库 → **Code** → **Download ZIP**（若解压后缺少 `database-full.sql`，请改用方式 A）。

> **不必** `git clone`，**不必** `git checkout` 任何分支。

### 第 2 步：进入文件夹

打开解压后的目录（项目根目录），确认能看到：

- `00-组长请看这里.txt`（快速上手）
- `pom.xml`
- `组长一键启动.vbs`（主启动入口，与本指南同目录）
- `docs\06-部署配置\database-full.sql`

文件位置总览见根目录 `**解压后目录说明.md`**。

### 第 3 步：双击启动


| 文件                   | 说明                                 |
| -------------------- | ---------------------------------- |
| `**组长一键启动.bat**`（推荐） | 输入 MySQL **root** 密码；自动清空并导入数据库 |
| `组长一键启动.cmd`         | 与 bat 相同（vbs 打不开时用）                |
| `组长一键启动.vbs`         | 仅转调 bat；若双击无反应请用 bat               |


脚本自动完成：检查环境 → **清空本地库** → 导入 `database-full.sql` → 启动后端 → 打开浏览器 **[http://localhost:8080](http://localhost:8080)**

**停止系统**：关闭标题为「CSRRM-后端服务」的窗口。

## 常见问题


| 现象                              | 处理                                                                                   |
| ------------------------------- | ------------------------------------------------------------------------------------ |
| 提示缺少 `database-full.sql` / 导入失败 | 重新 **Download ZIP**；再双击 bat（每次启动会清空并重新导入） |
| 后端窗口一闪就关 / `Malformed \uxxxx`   | 多为 `application-local.properties` 路径用了 `\`；重新运行 bat 生成配置（已自动改为 `/`）                  |
| 勿关「CSRRM-Backend」窗口             | 或在根目录执行 `mvnw.cmd spring-boot:run` 看完整报错                                             |
| `Access denied` / 没有权限连接 MySQL  | 填**本机 root 密码**（不是 admin123）；cmd 先测 `mysql -h 127.0.0.1 -u root -p`；已修复勿用共享库 init 脚本 |
| vbs 双击无反应                       | 改用 **组长一键启动.bat**；ZIP 解压后文件夹「解除锁定」                                                   |
| `mysql` 不是内部命令                  | 将 MySQL `bin` 加入系统 PATH                                                              |
| 页面空、无数据                         | 重新双击 bat（会自动清空并导入 SQL）；或手动运行 `.\scripts\import-database-local.ps1 -UseFullDump` |
| 8080 被占用                        | 见 `组长一键启动.bat` 说明或联系提交者                                                              |
