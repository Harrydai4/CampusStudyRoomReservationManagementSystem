# 组长复现指南（必读）

> **适用对象**：组长 / 验收老师 / 任意 clone 本仓库的同学  
> **目标**：在你电脑上打开的系统界面、数据库记录，与提交者本机一致。  
> **预计耗时**：首次约 15～30 分钟（含 Maven 下载依赖）。

---

## 0. 最简单：不用打开 PowerShell，双击启动

从 GitHub 下载并解压（或 `git clone`）后，进入**含 `pom.xml` 的项目根目录**：

| 双击这个文件 | 说明 |
|--------------|------|
| **`组长一键启动.vbs`**（推荐） | 弹出对话框输入 MySQL 密码；选是否导入数据库；自动完成其余步骤 |
| **`组长一键启动.bat`** | 同上，在黑色命令行窗口里按提示输入 |

脚本会自动：

1. 检查 Java、MySQL  
2. 尝试启动 MySQL 服务（`MySQL80`）  
3. 导入 `database-full.sql`（首次选「是」）  
4. 在新窗口启动后端  
5. 约 25 秒后打开 **http://localhost:8080**

**仍需事先安装**（无法通过双击自动安装）：**JDK 21+**、**MySQL 8**（并加入 PATH）。  
**不能自动提供的信息**：本机 MySQL `root` 密码 — 通过对话框或命令行问答输入。

登录系统（浏览器内，与 MySQL 无关）见本文 **第 4 节**。

> 可选：提交者运行 `.\scripts\build-leader-launcher-exe.ps1` 可生成 `dist\CSRRM-组长一键启动.exe`（实为解压包启动器，依赖同上）。

## 2. 一键复现（命令行方式，可选）

若已用 **第 0 节** 双击启动，可跳过本节。

在 **PowerShell** 中：

```powershell
.\scripts\setup-after-clone.ps1
```

脚本会自动完成：

1. 检查 Java、MySQL 客户端  
2. 生成本机 `application-local.properties`（**不会上传 Git**）  
3. 提示输入 **你自己电脑** 的 MySQL `root` 密码（与提交者可以不同）  
4. 导入 `docs/06-部署配置/database-full.sql`（覆盖为提交者快照数据）  
5. 关闭「启动时重置演示账号」，避免改掉已导入数据  
6. 执行 `mvnw spring-boot:run` 启动后端  
7. 约 25 秒后自动打开浏览器 **http://localhost:8080**

**停止服务**：在运行窗口按 `Ctrl + C`。

### 已知密码时（免交互）

```powershell
.\scripts\setup-after-clone.ps1 -MySqlPassword 你的密码
```

### 只导入、暂不启动

```powershell
.\scripts\setup-after-clone.ps1 -SkipStart
.\mvnw.cmd spring-boot:run
```

## 3. 分步操作（一键失败时用）

### 3.1 检查环境

```powershell
.\scripts\check-env.ps1
```

应看到 Java、MySQL 客户端为 OK，且「前端已构建」。

### 3.2 配置数据库密码

```powershell
Copy-Item src\main\resources\application-local.properties.example src\main\resources\application-local.properties
notepad src\main\resources\application-local.properties
```

修改一行（示例）：

```properties
spring.datasource.password=你的MySQL密码
```

建议追加（与提交者数据一致）：

```properties
app.upload.dir=D:/你的路径/CampusStudyRoomReservationManagementSystem/CSRRMupdate/uploads
app.demo.sync-accounts-on-startup=false
```

将路径改为你本机 **CSRRMupdate\uploads** 的绝对路径（斜杠 `/` 或 `\\` 均可）。

### 3.3 导入数据库

```powershell
.\scripts\import-database-local.ps1 -UseFullDump -Password 你的MySQL密码
```

成功提示：`imported database-full.sql`。

导入的是库名：**`study_room_reservation`**。

### 3.4 启动系统

```powershell
.\mvnw.cmd spring-boot:run
```

看到日志中出现 `Started CampusStudyRoomReservationManagementSystem` 后，浏览器打开：

**http://localhost:8080**

建议 **Ctrl + F5** 强制刷新。

## 7. 常见问题

### Q1：`Access denied for user 'root'@'localhost'`

本机 MySQL 密码与提交者不同，属正常。在 `application-local.properties` 或脚本参数中填写 **你自己的密码**。

### Q2：`mysqldump` / `mysql` 不是内部或外部命令

将 MySQL 的 `bin` 目录加入系统 PATH，例如：  
`C:\Program Files\MySQL\MySQL Server 8.0\bin`

### Q3：页面能开，但统计/列表为空

未导入 `database-full.sql`，或导入失败。重新执行 3.3，注意看报错。

### Q4：图片 404

`uploads/` 未 clone 完整，或 `app.upload.dir` 未指向项目内 `uploads` 目录。

### Q5：端口 8080 被占用

```powershell
$env:SERVER_PORT="8081"
.\mvnw.cmd spring-boot:run
```

浏览器访问 http://localhost:8081

### Q6：不支持发行版本 21

安装 [JDK 21+](https://adoptium.net/)，配置 `JAVA_HOME` 后重开 PowerShell。
