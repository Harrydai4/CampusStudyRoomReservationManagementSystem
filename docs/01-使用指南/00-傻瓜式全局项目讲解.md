## 第 0 章：先用一句话理解这个项目

**这是一个「校园自习室在线订座系统」**——就像订电影票选座位，只不过选的是图书馆/自习室的座位。

- **学生**用手机或电脑：注册 → 选日期时段和座位 → 到馆出示二维码 → 管理员扫码确认 → 学习 → 签退  
- **管理员**：审核新用户、管理自习室和座位、扫码签到、看统计报表

整个项目 = **网页（前端）+ 程序（后端）+ 数据库（MySQL）**，三部分缺一不可。

## 第 2 章：系统架构（分模块在做什么）

### 2.1 三层结构（必背）

```
┌─────────────────────────────────────────┐
│  ① 浏览器 + Vue 前端（frontend）         │  用户看到的页面
├─────────────────────────────────────────┤
│  ② Spring Boot 后端（src/main/java）     │  处理预约、签到、权限
├─────────────────────────────────────────┤
│  ③ MySQL 数据库                          │  永久保存用户、预约等
└─────────────────────────────────────────┘
```

**举例：学生点「确认预约」**

1. 前端把「哪个座位、什么时间」打包成 JSON，发给 `/api/reservations`
2. 后端检查：登录了吗？信用够吗？这座位这时段有没有人订了？
3. 都 OK → 往 MySQL 的 `reservation` 表插一条记录
4. 返回「成功」→ 前端弹提示并跳到签到页

### 2.2 后端内部模块（不用全记，知道去哪改）


| 模块   | 位置                                  | 干什么                 |
| ---- | ----------------------------------- | ------------------- |
| 接口   | `controller/`                       | 对外 URL，如登录、预约、上传    |
| 业务   | `service/AppService.java`           | 核心规则（最重要的大文件）       |
| 建表   | `service/DatabaseInitializer.java`  | 第一次启动自动建表、塞测试账号     |
| 定时任务 | `service/ScheduledTaskService.java` | 每分钟自动处理：没签到违约、到点签退等 |
| 安全   | `config/` + `support/`              | 登录 token、统一报错格式     |


### 2.3 前端内部（目前是一个大单文件）


| 文件           | 干什么           |
| ------------ | ------------- |
| `App.vue`    | 学生端 + 管理端所有页面 |
| `qr.js`      | 把签到码画成二维码     |
| `styles.css` | 手机端浅蓝卡片样式     |


### 2.4 数据库里主要存什么


| 表名（知道名字即可）                         | 存什么                    |
| ---------------------------------- | ---------------------- |
| `user_account` / `student_profile` | 学生账号、学院、信用分            |
| `admin_account`                    | 管理员                    |
| `study_room` / `seat`              | 自习室、每个座位               |
| `reservation`                      | 一条预约（谁、哪座、何时）          |
| `reservation_slot`                 | 10 分钟一片的时间格子（防两个人抢同一格） |
| `temp_leave`                       | 暂离记录                   |
| `operation_log`                    | 管理员操作日志                |

### 步骤 1：检查软件是否装好

在 PowerShell 里分别输入：

```powershell
java -version
mysql --version
node -v
npm -v
```


| 命令             | 期望                                  |
| -------------- | ----------------------------------- |
| `java`         | 显示 21 或更高                           |
| `mysql`        | 显示 8.x（或确认 Windows 服务里 MySQL80 在运行） |
| `node` / `npm` | 都有版本号                               |


**缺哪个装哪个**：

- Java：[https://adoptium.net/](https://adoptium.net/)  
- MySQL：[https://dev.mysql.com/downloads/installer/](https://dev.mysql.com/downloads/installer/)  
- Node：[https://nodejs.org/](https://nodejs.org/) （选 LTS）

---

### 步骤 2：配置 MySQL 密码（仅 root 有密码时需要）

若安装 MySQL 时设了 root 密码，启动后端前执行（把 `你的密码` 换成真实密码）：

```powershell
$env:DB_PASSWORD="你的密码"
```

默认配置是 root **空密码**，空密码则跳过本步。

---

### 步骤 3：启动后端（窗口 A）

```powershell
cd D:\SchoolWorkPlace\Database\CSRRMupdate
.\mvnw.cmd spring-boot:run
```

**第一次会较慢**（下载依赖）。成功标志：

```
Started CampusStudyRoomReservationManagementSystemApplication
```

若报 `Communications link failure` → MySQL 没开或密码错。  
若报 `不支持发行版本 21` → 换/装 JDK 21 并设置 `JAVA_HOME`。

此时可访问 [http://localhost:8080](http://localhost:8080) ，但界面可能是**旧打包版**。

---

### 步骤 4：安装并启动前端（窗口 B，推荐）

```powershell
cd D:\SchoolWorkPlace\Database\CSRRMupdate\frontend
npm install
npm run dev
```

浏览器打开：**[http://localhost:5173](http://localhost:5173)**

> 5173 会自动把 `/api` 请求转发到 8080，所以 **8080 和 5173 要同时开着**。

**若你只想用一个地址 8080**（例如答辩只开一个端口）：

```powershell
npm run build
```

然后回到窗口 A，Ctrl+C 停后端，再执行一次 `.\mvnw.cmd spring-boot:run`，访问 [http://localhost:8080](http://localhost:8080) 。

`npm run build` 若报权限错：

```powershell
node .\node_modules\vite\bin\vite.js build --outDir ..\src\main\resources\static --emptyOutDir
```

### 步骤 6：功能验收（打勾即完成必做）

#### 6.1 学生主流程

1. 学生登录 → 首页有公告、自习室
2. 点自习室 → **预约**：选日期、时间（10 分钟一档）、座位 → 确认
3. **签到页** → 点「出示二维码签到」→ 出现**方块二维码**（不是纯文字）
4. 管理员 → **签到管理** → 「扫码签到」或粘贴 token → 「手动确认」
5. 学生端状态变 **使用中**
6. 点 **暂离** → **暂离中** → **返回座位** → **使用中**
7. 点 **签退** → **已完成**

**扫码失败怎么办？** 复制学生二维码对应的 token 文本，粘贴到管理员签到页，点「手动确认」。

#### 6.2 资料与上传

1. 学生 → 我的 → **个人信息** → 改手机号 → 保存 → 刷新仍在
2. 管理员 → **自习室管理** → 编辑 → **选择图片上传** → 保存
3. 学生预约该室 → **座位分布图** 显示刚传的图

#### 6.3 管理端

1. **用户管理** → 通过 `202425220301`
2. **管理设置** → 能看到 **操作日志**
3. **统计分析** 能打开，CSV 能下载

全部打勾 = **V1.1 必做验收完成** ✅

## 第 6 章：常见问题速查


| 现象         | 处理                                      |
| ---------- | --------------------------------------- |
| 没有「暂离」按钮   | 用 5173 dev 模式，或重新 `npm run build` 后重启后端 |
| npm 不是命令   | 安装 Node.js，**重启 PowerShell**            |
| 登录 401/连不上 | 确认 8080 后端在跑；5173 时检查 vite 代理           |
| 上传图片 404   | 重启后端；看项目根目录是否生成 `uploads/layout/`       |
| 定时任务没反应    | 后端需连续运行 ≥1 分钟；见 README 可选验证             |
