# 07 — 多人共用一套系统与数据库（完整指南）

> **目标**：全组同学打开**同一个网址**，看到**同一批预约、签到、自习室数据**。  
> **推荐架构**：**一台电脑**跑 MySQL + Spring Boot，其他人**只开浏览器**。

---

## 1. 先搞懂：默认为什么各用各的数据？


| 组件          | 默认位置                  | 结果         |
| ----------- | --------------------- | ---------- |
| MySQL       | 每人本机 `localhost:3306` | 数据库互不相通    |
| Spring Boot | 每人本机 `8080`           | 只有本机浏览器能访问 |
| 上传图片        | 每人项目下 `uploads/`      | 图片不会同步     |


---

## 2. 三种角色（全组对齐）


| 角色              | 谁来做                  | 做什么                                    |
| --------------- | -------------------- | -------------------------------------- |
| **A. 服务器管理员**   | 1 人（宿舍/实验室一台性能稳定的电脑） | 启动共用 MySQL + Spring Boot，告知 IP         |
| **B. 普通组员**     | 其余所有人                | 浏览器访问 `http://服务器IP:8080`，**不要**各自再跑后端 |
| **C. 开发调试（可选）** | 需要改代码的人              | 本机跑后端，但 `DB_URL` 指向共用 MySQL（见第 6 节）    |


**答辩最稳**：只采用 **A + B**，全组访问一个地址。

---

## 3. 方案一（推荐）：一台 Windows 电脑当「应用 + 数据库」服务器

### 3.1 环境要求（服务器管理员）

- JDK 21+
- Docker Desktop **或** 本机 MySQL 8
- 项目代码 `CSRRMupdate` 已克隆到服务器

### 3.2 启动共用 MySQL

**方式 A — Docker（推荐，免装 MySQL）**

```powershell
cd D:\SchoolWorkPlace\Database\CSRRMupdate
.\scripts\setup-shared-mysql-docker.ps1
```

默认账号：


| 项       | 值                        |
| ------- | ------------------------ |
| 数据库     | `study_room_reservation` |
| 用户      | `study`                  |
| 密码      | `csrrm_shared_123`       |
| root 密码 | `csrrm_root_123`         |
| 端口      | `3306`                   |


**方式 B — 本机已装 MySQL**

在**管理员**命令行执行（或用 Navicat 运行 SQL 文件）：

```powershell
mysql -u root -p < docs\06-部署配置\init-shared-mysql.sql
```

脚本会创建库 `study_room_reservation` 和用户 `study`（密码 `csrrm_shared_123`）。**部署后请改强密码。**

### 3.3 配置共用 Spring Boot

```powershell
cd D:\SchoolWorkPlace\Database\CSRRMupdate
Copy-Item src\main\resources\application-shared.properties.example src\main\resources\application-shared.properties
notepad src\main\resources\application-shared.properties
```

至少修改：

```properties
# MySQL 与应用在同一台机器时用 127.0.0.1
spring.datasource.url=jdbc:mysql://127.0.0.1:3306/study_room_reservation?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true&useSSL=false
spring.datasource.username=study
spring.datasource.password=csrrm_shared_123

# 上传目录（绝对路径，与项目在同一台服务器）
app.upload.dir=D:/SchoolWorkPlace/Database/CSRRMupdate/uploads
```

> `application-shared.properties` 已在 `.gitignore`，**不要提交 Git**。

### 3.4 测试数据库连接

```powershell
.\scripts\test-db-connection.ps1 -Password csrrm_shared_123
```

### 3.5 放行防火墙（管理员 PowerShell）

```powershell
.\scripts\open-firewall-shared.ps1
```

### 3.6 启动共用 Web 服务

```powershell
.\scripts\start-shared-server.ps1
```

脚本会显示类似：

```text
组员浏览器访问：http://172.22.173.26:8080
```

### 3.7 组员怎么访问？

1. 手机/电脑连接**与服务器同一 WiFi**
2. 浏览器打开：`http://<服务器IPv4>:8080`
3. 演示账号（每次重启后端会自动恢复）：


| 角色  | 账号           | 密码       |
| --- | ------------ | -------- |
| 学生  | 202301010101 | 123456   |
| 管理员 | admin        | admin123 |


1. **Ctrl+F5** 强刷，避免缓存旧页面

## 4. 方案二：数据库与应用分机（进阶）


| 机器   | 跑什么                                                                        |
| ---- | -------------------------------------------------------------------------- |
| 机器 1 | 仅 MySQL（Docker 或独立安装），放行 **3306**                                          |
| 机器 2 | Spring Boot，`application-shared.properties` 里 `127.0.0.1` 改为 **机器 1 的 IP** |


```properties
spring.datasource.url=jdbc:mysql://192.168.1.50:3306/study_room_reservation?...
```

应用机器仍需配置 `app.upload.dir` 为**本机绝对路径**（上传文件存在应用服务器上）。

## 5. 方案三：云数据库 RDS（有公网 IP 时）

```properties
spring.datasource.url=jdbc:mysql://rm-xxxx.mysql.rds.aliyuncs.com:3306/study_room_reservation?useSSL=true&serverTimezone=Asia/Shanghai&...
spring.datasource.username=study
spring.datasource.password=强密码
```

- 云控制台安全组放行应用服务器 IP
- **不要**对 `0.0.0.0/0` 开放 MySQL 公网（除非课程明确要求且设强密码）

## 6. 开发同学本机连远程库（可选，不推荐答辩时用）

若你本机也要跑 `spring-boot:run`，但数据要写入共用库：

**方式 A — 配置文件**

```properties
# application-shared.properties
spring.datasource.url=jdbc:mysql://192.168.1.100:3306/study_room_reservation?...
```

**方式 B — 当前 PowerShell 窗口环境变量**

```powershell
$env:DB_URL="jdbc:mysql://192.168.1.100:3306/study_room_reservation?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true&useSSL=false"
$env:DB_USERNAME="study"
$env:DB_PASSWORD="csrrm_shared_123"
$env:UPLOAD_DIR="D:\SchoolWorkPlace\Database\CSRRMupdate\uploads"
.\mvnw.cmd spring-boot:run
```

注意：多人同时跑多个 Spring Boot 连同一库时，**上传图片仍各存本机 uploads**，只有访问**服务器那台**的后台才能看到全部图片。答辩仍建议**只跑一个后端**。

## 7. 备份与恢复

### 7.1 备份数据库

```powershell
.\scripts\backup-shared-db.ps1 -Password csrrm_shared_123
```

生成文件在 `backups/study_room_reservation-时间戳.sql`。

### 7.2 备份上传文件

复制整个 `uploads/` 目录（与数据库同一台应用服务器）。

### 7.3 恢复（示例）

```powershell
mysql -u study -p study_room_reservation < backups\study_room_reservation-xxxx.sql
```

## 8. 配置文件对照


| 文件                                      | 用途     | 是否提交 Git |
| --------------------------------------- | ------ | -------- |
| `application.properties`                | 公共默认项  | ✅        |
| `application-local.properties`          | 本机单人开发 | ❌        |
| `application-shared.properties`         | 共用服务器  | ❌        |
| `application-shared.properties.example` | 共用配置模板 | ✅        |


环境变量（可覆盖配置文件）：


| 变量                            | 含义                   |
| ----------------------------- | -------------------- |
| `DB_URL`                      | JDBC 连接串             |
| `DB_USERNAME` / `DB_PASSWORD` | 数据库账号                |
| `UPLOAD_DIR`                  | 上传目录绝对路径             |
| `SERVER_ADDRESS`              | 默认 `0.0.0.0`（局域网可访问） |
| `SERVER_PORT`                 | 默认 `8080`            |
| `DB_POOL_SIZE`                | 连接池大小，默认 15          |

## 9. 常见问题


| 现象                            | 处理                                                        |
| ----------------------------- | --------------------------------------------------------- |
| 组员打不开 `http://IP:8080`        | 服务器防火墙执行 `open-firewall-shared.ps1`；确认同一 WiFi             |
| `Communications link failure` | MySQL 未启动；`test-db-connection.ps1` 测端口                    |
| `Access denied`               | 密码错；检查 `application-shared.properties`                    |
| 数据不一致                         | 有人还在本机跑后端连本机库；统一只访问服务器                                    |
| 图片 404                        | 图片在**应用服务器**的 `uploads/`；不要多人各跑一个后端                       |
| 改密后无法登录                       | 重启服务器后端（`app.demo.sync-accounts-on-startup=true` 会恢复演示密码） |
| Docker MySQL 首次启动失败           | 3306 被占用；`netstat -ano                                    |

## 10. 一条龙命令（服务器管理员复制）

```powershell
cd D:\SchoolWorkPlace\Database\CSRRMupdate

# 1. Docker 共用 MySQL（若无 Docker 则跳过，改用手动 init-shared-mysql.sql）
.\scripts\setup-shared-mysql-docker.ps1

# 2. 首次配置（编辑密码与 upload 路径）
Copy-Item src\main\resources\application-shared.properties.example src\main\resources\application-shared.properties
notepad src\main\resources\application-shared.properties

# 3. 测库 + 防火墙（防火墙需管理员 PowerShell）
.\scripts\test-db-connection.ps1 -Password csrrm_shared_123
.\scripts\open-firewall-shared.ps1

# 4. 启动共用 Web（保持窗口不关）
.\scripts\start-shared-server.ps1
```

把终端里打印的 `http://x.x.x.x:8080` 发给全组即可。

## 13. 同学不在同一 WiFi？

局域网 IP（`http://192.168.x.x:8080`）**只在同一 WiFi 有效**。

跨网络、仍要共用同一套网页与数据库，请改用：

→ **[08-跨WiFi共用与安全访问.md](08-跨WiFi共用与安全访问.md)**  
（Tailscale 私有组网 **或** Cloudflare HTTPS 隧道；只暴露 8080，不裸奔整台电脑）
