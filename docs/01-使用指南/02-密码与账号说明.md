# 02 — 密码与账号说明（一览表）

> **重要**：下面「MySQL 密码」是你电脑上的数据库密码，**不要提交到 Git**。  
> 「系统登录密码」在数据库 SQL 里，已随 `database-full.sql` 一起交给组长。

---

## 1. MySQL 数据库密码（本机环境）

| 用途 | 你的值（当前） | 配置位置 |
|------|----------------|----------|
| 本机 root 连接 MySQL | **123456** | `src/main/resources/application-local.properties` → `spring.datasource.password` |
| 环境变量（可选，覆盖上面） | 同左 | PowerShell：`$env:DB_PASSWORD="123456"` |
| 导出 SQL 脚本 | 同左 | `.\scripts\export-database-for-git.ps1 -Password 123456` |
| 导入 SQL 脚本 | 同左 | `.\scripts\import-database-local.ps1 -Password 123456` |
| 一键验收脚本 | 同左 | `.\scripts\setup-after-clone.ps1 -MySqlPassword 123456` |

**首次配置**：复制 `application-local.properties.example` → `application-local.properties`，把密码改成 `123456`。

**组长电脑**：密码可以和你不同，他只在**自己本机**的 `application-local.properties` 里填他的 MySQL 密码即可。

## 3. 系统登录账号（在 SQL 里，不是 MySQL 密码）

导入 `database-full.sql` 后，浏览器 **http://localhost:8080** 使用：

| 角色 | 账号 | 登录密码 |
|------|------|----------|
| 学生 | `202301010101` | `123456` |
| 学生 | `202301010102` | `123456` |
| 管理员 | `admin` | `admin123` |
| 超级管理员 | `superadmin` | `super123` |

你在测试里新建的管理员/学生，以 **SQL 快照里为准**（例如可能有 `Ladmin` 等账号，密码为创建时设置的 BCrypt，默认新建管理员常用 `admin123`）。

## 5. 不要提交到 Git 的文件

---

## 6. 和「数据库有没有提交完整」的关系

| 内容 | 是否在 Git 里 |
|------|----------------|
| 表结构 + 全部业务数据 | ✅ `docs/06-部署配置/database-full.sql`（mysqldump 导出） |
| 上传图片 | ✅ `uploads/` |
| MySQL 的 root 密码 **123456** | ❌ 不应提交（每人本机自己配） |

若 `git push` 成功，组长导入 SQL 后数据与你导出时一致；他只需自己配 MySQL 密码，不需要和你一样是 `123456`。
