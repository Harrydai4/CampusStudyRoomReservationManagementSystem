# 01 — 必做事项与分工说明

> **给谁看**：想搞清楚「哪些必须我做」「哪些助手已经做完」「哪些是可选加分」  
> **工作目录**：`D:\SchoolWorkPlace\Database\CSRRMupdate`

---

## 一句话结论

**真正「只有你能做、且不做就跑不起来」的，通常只有 2～3 项：安装并启动 MySQL、（如有密码）配置数据库密码、启动后端做一次功能验收。**

其余很多原先写在「助手无法代劳」里的事项，**助手已在你的机器上完成或已提供脚本/模板**；课程 Word 报告仍需你在 Word 里编辑。

---

## 四类分工（对照表）


| 类别                 | 含义                      | 你要做什么                                             |
| ------------------ | ----------------------- | ------------------------------------------------- |
| **A. 必做 · 仅用户**    | 助手无法替你操作本机服务或改 Word 二进制 | 安装/启动 MySQL、填密码、写课程报告                             |
| **B. 必做 · 助手已完成**  | 启动前环境或构建，助手已在本仓库交付      | 你只需 **启动后端 + 浏览器验收**                              |
| **C. 必做 · 需你执行一步** | 助手写好命令，需你在 PowerShell 跑 | 见 [05-启动流程与PowerShell命令](05-启动流程与PowerShell命令.md) |
| **D. 可选 · 加分/答辩**  | 不做也能完成 V1.1 演示          | 手机真机、Nginx、Docker 等                               |

## 你现在最少要做的事（最短路径）

1. **启动 MySQL**（本机服务或 `docker compose up -d`，见 `docs/06-部署配置/`）
2. **（可选）** 若 root 有密码：设置 `$env:DB_PASSWORD="你的密码"`
3. **启动后端**（完整步骤见 [05-启动流程与PowerShell命令.md](05-启动流程与PowerShell命令.md)）：

```powershell
cd D:\SchoolWorkPlace\Database\CSRRMupdate
.\mvnw.cmd spring-boot:run
```

1. 浏览器打开 **[http://localhost:8080](http://localhost:8080)**（前端已打包进后端，无需再开 5173）
2. 用测试账号走一遍 [验收清单](../../README.md#验收清单v11)

环境自检（可选）：

```powershell
.\scripts\check-env.ps1
```

## 常见误解


| 误解               | 实际情况                                |
| ---------------- | ----------------------------------- |
| 「助手无法代劳 = 全部必做」  | ❌ 很多是**环境安装**或**可选演示**，M4/M6 等助手已完成 |
| 「必须开两个窗口跑前后端」    | ❌ 前端已 build 时，**只开后端 + 访问 8080** 即可 |
| 「必须手机扫码才能答辩」     | ❌ 管理员端可粘贴 token；手机同 WiFi 只是加分演示     |
| 「必须手插 SQL 测定时任务」 | ❌ 运行 `mvnw test` 已覆盖未签到违约逻辑         |
