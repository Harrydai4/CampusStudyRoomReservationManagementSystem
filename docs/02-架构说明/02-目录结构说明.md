# 目录结构说明

> 本文说明 `CSRRMupdate` 根目录下**每个文件夹/关键文件**的职责。  
> 原则：**代码在 `src/` 与 `frontend/`，文档在 `docs/`，运行时数据在 `uploads/`。**

---

## 根目录一览

```
CSRRMupdate/
├── docs/                    # 📚 全部文档（不参与编译）
├── scripts/                 # 🔧 环境检查、前端构建（check-env / build-frontend）
├── frontend/                # 🖥️ Vue 3 前端源码
├── src/                     # ☕ Spring Boot 后端源码 + 打包后的静态页
├── uploads/                 # 📎 运行时上传文件（git 忽略，本地自动生成）
├── target/                  # 🔧 Maven 编译产物（git 忽略）
├── .mvn/                    # Maven Wrapper 配置
├── pom.xml                  # 后端依赖与构建配置
├── mvnw / mvnw.cmd          # 无需全局安装 Maven 即可构建
├── README.md                # 项目入口：启动与验收
└── .gitignore               # Git 忽略规则
```

---

## `docs/` — 文档中心


| 子目录             | 内容                  |
| --------------- | ------------------- |
| `01-使用指南/`       | 面向新手的讲解、操作指南        |
| `02-架构说明/` | 架构图、目录规范            |
| `03-开发维护/`  | 开发原则、问题记录           |
| `04-版本记录/`    | 版本基线、更新说明、更新日志      |
| `05-需求与设计/`         | 需求/概要/详细设计（课程正式文档）  |
| `06-部署配置/`         | Docker Compose、Nginx 示例   |
| `07-原型资源/`    | HTML 原型（参考用，非运行时代码） |


**规范**：新增说明类 `.md` 一律放入 `docs/` 对应子目录，**不要在根目录堆文档**。

---

## `frontend/` — 前端工程

```
frontend/
├── src/
│   ├── App.vue          # 主应用（学生端 + 管理端全部页面）
│   ├── main.js          # Vue 入口
│   ├── qr.js            # 二维码 SVG 生成
│   └── styles.css       # 全局样式
├── index.html           # HTML 壳
├── vite.config.js       # 开发服务器 + /api 代理到 8080
├── package.json         # npm 依赖与脚本
└── package-lock.json
```


| 命令              | 作用                                                    |
| --------------- | ----------------------------------------------------- |
| `npm install`   | 安装依赖（首次必做）                                            |
| `npm run dev`   | 开发模式 → [http://localhost:5173](http://localhost:5173) |
| `npm run build` | 打包到 `../src/main/resources/static`                    |


---

## `src/` — 后端工程

```
src/main/java/com/scau/campusstudyroomreservationmanagementsystem/
├── CampusStudyRoomReservationManagementSystemApplication.java  # 启动类
├── controller/          # REST 接口（对外 URL）
│   ├── AppController.java
│   └── UploadController.java
├── service/             # 业务逻辑
│   ├── AppService.java
│   ├── DatabaseInitializer.java
│   └── ScheduledTaskService.java
├── config/              # 安全、JWT、静态资源
│   ├── SecurityConfig.java
│   ├── JwtAuthFilter.java
│   ├── JwtService.java
│   └── WebConfig.java
└── support/             # 统一响应、异常、当前用户

src/main/resources/
├── application.properties   # 端口、数据库、JWT、上传目录
└── static/                  # 前端 build 产物（8080 直接访问）

src/test/java/             # 测试（当前仅启动上下文测试）
```

**规范**：

- 新接口 → `controller/`
- 新业务流程 → `service/`（或新建 XxxService）
- 不改 `support/` 除非调整统一响应/异常格式

---

## `uploads/` — 用户上传文件

```
uploads/
└── layout/              # 自习室座位分布图
    └── xxxxx.jpg
```

通过 URL `/uploads/layout/xxxxx.jpg` 访问。目录在首次上传或启动后自动创建。

---

## 不应提交到 Git 的内容

已在 `.gitignore` 中配置：

- `target/` — Maven 编译输出
- `frontend/node_modules/` — 前端依赖
- `uploads/` — 本地上传文件
- `.idea/`、`.vscode/` — IDE 配置
