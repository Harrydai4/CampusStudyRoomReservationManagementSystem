## 2. 新同学 clone 后如何导入（推荐）

在项目根目录 PowerShell：

```powershell
cd D:\SchoolWorkPlace\Database\CSRRMupdate

# 方式 A：一键脚本（读 application-local.properties 里的密码）
.\scripts\import-database-local.ps1

# 方式 B：只导入合一文件
.\scripts\import-database-local.ps1 -UseFullDump

# 方式 C：手动（root 有密码时）
mysql -u root -p < docs\06-部署配置\init-shared-mysql.sql
mysql -u root -p study_room_reservation < docs\06-部署配置\schema.sql
mysql -u root -p study_room_reservation < docs\06-部署配置\data.sql
```

然后启动：

```powershell
Copy-Item src\main\resources\application-local.properties.example src\main\resources\application-local.properties
# 编辑密码后
.\mvnw.cmd spring-boot:run
```

浏览器 **http://localhost:8080**

## 4. 维护人员：从本机重新导出 SQL

改表结构或演示数据后，在**已跑通**的机器上执行：

```powershell
.\scripts\export-database-for-git.ps1
git add docs/06-部署配置/schema.sql docs/06-部署配置/data.sql docs/06-部署配置/database-full.sql
git commit -m "chore(db): refresh schema and seed data"
```

## 5. 签到方式（V1.2.1）

管理员在管理端「签到」页输入**学生学号** →「确认签到」。  
学生端在签到页显示学号并**等待管理员操作**，无需二维码/token。  
API：`POST /api/admin/checkin/scan` body `{ "studentNo": "202301010101" }`

## 6. 与 Java 自动建表的关系

| 方式 | 何时用 |
|------|--------|
| 导入 `schema.sql` + `data.sql` | 课设提交、clone 验收、与组长库一致 |
| 仅 `spring-boot:run` | 日常开发；`DatabaseInitializer.java` 会 `create table if not exists` |

两种方式可并存；**V1.2 起 GitHub 必须包含上述 SQL 文件**。
