# 课设数据库交付清单（必须随 GitHub 一并提交）

> **说明**：数据库课设除 Java 代码外，必须提交**可独立导入**的 SQL 文件。  
> 仅 clone 代码、不导入 SQL，MySQL 里只有空库或没有演示数据，老师/组长无法验收。

---

## 1. 本目录必须存在的文件

| 文件 | 用途 | Git 提交 |
|------|------|:---:|
| [schema.sql](schema.sql) | 全部表结构、索引、视图 | ✅ 必须 |
| [data.sql](data.sql) | 全部演示/业务数据（INSERT） | ✅ 必须 |
| [database-full.sql](database-full.sql) | 结构+数据合一（网盘/一键导入） | ✅ 必须 |
| [init-shared-mysql.sql](init-shared-mysql.sql) | 建库 + 共用账号 | ✅ 必须 |
| [数据库文件说明.md](数据库文件说明.md) | 导入步骤 | ✅ 必须 |
| 本文件 `课设数据库交付清单.md` | 交付对照 | ✅ 必须 |

**禁止提交**：`application-local.properties`、含真实密码的备份、`backups/*.sql`（带时间戳的运行时备份）。

---

## 2. 一键刷新 SQL（开发机有数据时）

```powershell
cd D:\SchoolWorkPlace\Database\CSRRMupdate
.\scripts\export-database-for-git.ps1
git add docs/06-部署配置/schema.sql docs/06-部署配置/data.sql docs/06-部署配置/database-full.sql
git commit -m "chore(db): refresh schema and seed data"
git push
```

## 3. 验收方 clone 后（推荐一键）

```powershell
.\scripts\setup-after-clone.ps1
```

或分步：

```powershell
.\scripts\import-database-local.ps1 -UseFullDump
.\mvnw.cmd spring-boot:run
```

详见 [09-全量交付与组长一键验收.md](../01-使用指南/09-全量交付与组长一键验收.md)。

验证：

```sql
USE study_room_reservation;
SHOW TABLES;
SELECT COUNT(*) FROM study_room;
SELECT student_no, name FROM student_profile LIMIT 5;
```
