## 一、本次实施目标回顾

本次变更以"第三版需求文档数据字典"为硬约束，对整个系统（后端运行时初始化、SQL 快照、前端展示、测试夹具）进行了一次系统性的规范化落地。核心原则：

1. **枚举值直接在数据库中存中文**，不再存英文代码
2. **信用分统一采用 0-500 口径**
3. **数据库规范化**：拆多值字段、删物理派生字段、补外键约束
4. **数据清洗**：学号 12 位、年级 4 位、预约号 16 位、房间/座位编号 ≤10 字符
5. **前后端一致性**：后端、SQL 快照、前端展示三方对齐

### 2.2 前端（1 个 Vue 文件 + 2 个 CSS 文件）

#### `frontend/src/App.vue`
**新增统一归一化层**：避免直接用英文码值判断业务状态，实现"后端中文枚举 ↔ 前端中文展示"的双向兼容。

```
canonical(value, MAP) → 根据 MAP 将任何历史码值归一化为中文
```
| 归一化函数 | MAP 名称 | 映射内容 |
|-----------|---------|---------|
| `reservationStatusValue()` | `RES_STATUS_MAP` | `PENDING→待使用, USING→使用中, COMPLETED→已完成, CANCELLED→已取消, VIOLATED→已违约` |
| `accountStatusValue()` | `ACCOUNT_STATUS_MAP` | `NORMAL→正常, DISABLED→禁用, BLACKLIST→黑名单, PENDING→待审核` |
| `adminStatusValue()` | `ADMIN_STATUS_MAP` | `NORMAL→正常, DISABLED→离职` |
| `adminRoleValue()` | `ADMIN_ROLE_MAP` | `SUPER_ADMIN→超级管理员, ADMIN→普通管理员` |
| `roomStatusValue()` | `ROOM_STATUS_MAP` | `OPEN→开放, CLOSED→关闭, MAINTENANCE→维护中` |
| `seatStatusValue()` | `SEAT_STATUS_MAP` | `NORMAL→空闲, DAMAGED→维修, DISABLED→停用` |
| `feedbackStatusValue()` | `FEEDBACK_STATUS_MAP` | `PENDING→待处理, DONE→已处理` |
| `auditStatusValue()` | `AUDIT_STATUS_MAP` | `PENDING→待审核, APPROVED→已通过, REJECTED→已拒绝` |

**覆盖的旧判断点**（约 36 处调用）：
- 签到页面状态判断：`isPendingReservation()` / `isUsingReservation()`
- 预约页面筛选：`reservationTabs = [{ key: '待使用', label: '待使用' }, ...]`
- 座位启停：`seatStatusValue(s.status) === '空闲' ? '停用' : '空闲'`
- 预约卡片状态展示：`statusText()`
- 统计页违约计数：`reservationStatusValue(r.status) === '已违约'`
- 管理员筛选器：`adminStatusFilter / roomStatusFilter / seatStatusFilter / feedbackStatusFilter`
- 今日预约/进行中预约过滤：`['待使用', '使用中'].includes(reservationStatusValue(r.status))`
- 注册年级处理：直接使用 4 位数字字符串（如 `"2023"`），展示时显示"2023 级"
