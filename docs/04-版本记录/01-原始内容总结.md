# 01 — 原始内容总结（复制基线 V1.0）

> 本文描述将工程复制到 `CSRRMupdate` 时的**初始状态**，作为 V1.1 更新的对照基线。  
> 总结日期：2026-05-25

---

## 1. 项目定位

**校园自习室预约管理系统**，华南农业大学软件工程 8 班数据库课程设计项目（文档编号 SCAU-SE-08-G03）。

- **学生端**：注册/登录、预约、签到签退、信用、统计、公告、通知、反馈  
- **管理端**：用户审核、自习室/座位、预约、扫码签到、公告、统计、反馈  
- **架构**：B/S，Spring Boot + Vue 3 + MySQL

## 3. 基线已完成能力

### 3.1 后端（约 75%）

- **40+ REST API**：认证、自习室、座位、预约、签到、签退、信用、统计、公告、通知、反馈、管理端 CRUD  
- **13 张数据表**：`user_account`、`student_profile`、`admin_account`、`study_room`、`seat`、`reservation`、`reservation_slot`、`checkin_record`、`credit_log`、`blacklist_record`、`announcement`、`notification_message`、`feedback_ticket`  
- **启动初始化**：`DatabaseInitializer` 建表 + 种子账号  
- **业务规则**：10 分钟时间片冲突检测、JWT 鉴权、信用积分变动、CSV 导出

### 3.2 前端（约 70%）

- 学生端全页面 + 管理端九大模块  
- 真实 QR 码（`qr.js`）、管理员摄像头扫码（`BarcodeDetector`）  
- 移动端浅蓝卡片 UI，预约 10 分钟步长

### 3.3 文档（约 95%）

- 需求/概要/详细设计完整  
- `问题记录.md` 记录 10 项 UI/联调修复

## 6. 基线已知环境限制（问题记录继承）

1. 项目需 **Java 21** 编译（系统默认 JDK 20 会报错）
2. Windows 下 `npm run build` 可能权限失败，可用 `node node_modules/vite/bin/vite.js build` 替代
3. 管理员扫码依赖浏览器 `BarcodeDetector`，部分手机需手动粘贴 token
