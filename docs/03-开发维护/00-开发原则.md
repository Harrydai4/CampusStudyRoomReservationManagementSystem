# 校园自习室预约管理系统 — 开发原则

> **工作目录（唯一开发根目录）**：`D:\SchoolWorkPlace\Database\CSRRMupdate`  
> **版本基线**：V1.1 更新版  
> **最后更新**：2026-05-25

---

## 1. 唯一工作区原则

1. **所有开发、文档编写、构建与测试，均在本文件夹 `CSRRMupdate` 内完成。**
2. 不得回写 `CampusStudyRoomReservationManagementSystem-master`、`HTML` 等兄弟目录；它们仅作历史参考。
3. 每次迭代完成后，在 `[docs/04-版本记录/03-更新日志.md](../04-版本记录/03-更新日志.md)` 记录「计划更新」与「实际完成」。

---

## 2. 目录规范


| 类型      | 放置位置                      | 禁止           |
| ------- | ------------------------- | ------------ |
| 源代码（后端） | `src/main/java/...`       | 根目录堆 `.java` |
| 源代码（前端） | `frontend/src/...`        | 根目录堆 `.vue`  |
| 文档      | `docs/` 对应子目录             | 根目录堆大量 `.md` |
| 上传文件    | `uploads/`                | 提交到 Git      |
| 编译产物    | `target/`、`node_modules/` | 提交到 Git      |


详见 [目录结构说明](../02-架构说明/02-目录结构说明.md)。

---

## 3. 需求对齐原则

1. 功能边界以 `[docs/05-需求与设计/概要设计与详细设计.md](../05-需求与设计/概要设计与详细设计.md)` 为准。
2. 交互参考 `[docs/07-原型资源/原型设计.html](../07-原型资源/原型设计.html)`，不引入设计文档未描述的业务。
3. 新增功能须先写入 `[03-更新日志.md](../04-版本记录/03-更新日志.md)` 的「计划更新」，完成后再移入「实际完成」。

---

## 4. 技术实现原则


| 项   | 约定                                                                |
| --- | ----------------------------------------------------------------- |
| 后端  | Spring Boot 3.5 + JdbcTemplate；新增代码需有必要的中文注释                      |
| 前端  | Vue 3 单页 `frontend/src/App.vue`；构建输出到 `src/main/resources/static` |
| 数据库 | MySQL 8；`DatabaseInitializer` 启动建表                                |
| 鉴权  | JWT + Spring Security；管理端按 `manager_id` 校验                        |
| 文件  | 上传目录 `uploads/`；HTTP 路径 `/uploads/`**                             |
