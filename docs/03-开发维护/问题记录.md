## 1. 学生端生成签到二维码没有实现

**问题**：学生端点击“生成签到二维码”后，页面只是展示 token 的前 32 位文本，并不是真正可被扫码识别的二维码；二维码内容也没有明确绑定学生信息。

**原因**：前端使用 `.fake-qr` 样式伪造二维码视觉效果，没有调用二维码编码逻辑。后端 `/api/checkin/qrcode` 只返回简单 token，没有把学生学号、姓名和预约展示信息一起返回给前端。

**解决思路**：后端根据当前登录学生、待签到预约、生成时间、学号和姓名生成签到 token，并返回学生和预约信息；前端把 token 编码成真实 QR SVG，同时在二维码下展示学生信息和预约位置，便于管理员核对。

**具体解决方法**：新增 `frontend/src/qr.js` 的 `createQrSvg`；`App.vue` 的 `loadQr()` 生成 SVG；`AppService.qrCode()` 返回 `qrToken`、姓名、学号、自习室、座位、60 秒有效期。

**验收状态：✅ 已修复**

- `frontend/src/qr.js` 存在且被 `loadQr()` 调用  
- 弹窗展示 `qrSvg` + `qrInfo`（姓名、学号、房间、座位）  
- 无 `.fake-qr` 样式残留

## 2. 管理员端手机签到需要调用摄像头扫码

**问题**：管理员端签到管理只支持粘贴二维码 token，手机使用时无法直接调用摄像头扫描学生二维码完成签到。

**原因**：前端签到管理页没有 `getUserMedia` 摄像头调用，也没有二维码识别循环；后端扫码接口虽然存在，但前端没有把摄像头识别结果自动提交给接口。

**解决思路**：在管理员签到页增加“打开摄像头扫码”入口。移动端通过 `getUserMedia` + `BarcodeDetector` 识别后提交 `/api/admin/checkin/scan`；保留手动粘贴兜底。

**具体解决方法**：`App.vue` 管理员签到区 `startCameraScan` / `scanCameraFrame` / `stopCameraScan`；`AppService.scanCheckin()` 校验 token 60 秒有效。

**验收状态：✅ 已修复**

- 管理端「扫码签到/停止扫码」+ 视频预览 + `scanner-*` 样式  
- 识别成功自动 `scanCheckin()`  
- 不支持识别时仍可手动粘贴 token

## 3. Maven 测试使用默认 JDK 时版本不匹配

**问题**：直接运行 `mvnw test` 时，默认系统 Java 是 20，而项目 `pom.xml` 配置 `java.version` 为 21，导致 Maven 编译报“不支持发行版本 21”。

**原因**：项目要求 Java 21 编译，但系统 PATH 中优先使用的是较低版本 JDK。

**解决思路**：安装 JDK 21+ 并设置 `JAVA_HOME`；或使用项目内 `.tools/jdk21`（若已配置）。

**具体解决方法**：业务代码无需改；执行前设置 `JAVA_HOME` 指向 JDK 21。

**验收状态：✅ 已规避（环境）**

- 本机 `java -version` 为 21+ 时 `.\mvnw.cmd test` 通过  
- 见 `docs/01-使用指南/05-启动流程与PowerShell命令.md`
